\begin{frame}{Adaptation}

To transition from trees to flat sets, values must be moved from nodes directly into descriptors. It should consider three phases:

\begin{itemize}
    \item \textbf{Acquire:} Thread installs its descriptor $D$ into target cells $R_j$ via CAS in ascending address order (prevents deadlocks). If blocked by descriptor $U$, the thread helps $U$ finish first.

    \pause

    \item \textbf{Decision (Linearization Point):} Once all cells are acquired, the thread atomically changes descriptor status via CAS from \textit{Undecided} to \textit{Committed} or \textit{Aborted}.

    \pause

    \item \textbf{Release:} Thread resets cell states to concrete values. Updates to $new_j$ if \textit{Committed}, or reverts to $old_j$ if \textit{Aborted}, exposing the changes to the system.
\end{itemize}

\pause

Since the specification draws no difference between threads and their descriptors, all updates occur concurrently.

\end{frame}


\begin{frame}{Algorithm Specifications}

State variables:

\begin{itemize}
    \item \texttt{wDescState} Descriptor status (\textit{IDLE, ACTIVE, SCANNING, SUCCESS, ABORTED}).
    \item \texttt{wDescPending} Cells awaiting locking or physical update.
    \item \texttt{wDescFrame} Keys (cells) acquired by the transaction.
    \item \texttt{wDescOldValue} Snapshot of original cell values ($old_j$).
    \item \texttt{wDescNewValue} Snapshot of target cell values ($new_j$).
\end{itemize}

\end{frame}
